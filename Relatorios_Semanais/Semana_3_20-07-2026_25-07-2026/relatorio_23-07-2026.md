## Hoje, dia 23/07/2026, criei um pequeno script .ps1 para mostrar a temperatura hora a hora do dia, ou a temperatura atual, com ajuda da API gratuita da página open-meteo.

### 1- Encontrei a página web ***open-meteo.com***, que conta com uma opcao para selecionar a localizacao e zona horaria desejada, assim como tambem uma api completamente gratuita. É open source.
### 2- Configurei a página e a URL da api para mostrar a temperatura horária do dia atual e a temperatura da hora atual.
### 3- Após pesquisas, o comando *Invoke-RestMethod* pareceu-me melhor do que o comando *Invoke-WebRequest*, pois converte automáticamente a resposta Json da url (***https://api.open-meteo.com/v1/forecast?latitude=40.6617&longitude=-7.909&hourly=temperature_2m&timezone=auto&forecast_days=1&current=temperature_2m***, neste caso) para um objeto powershell. Sem ele, seria necessário converter o output para um objeto powershell com outro comando (*ConvertFrom-Json*) para melhor formatacao.
### 4- Investiguei alguns comandos úteis na página de Microsoft e encontrei o comando *Read-Host*, que permite ler o input na consola. Juntei com outro comando novo que descobri na pesquisa, *switch(case){}*, e escrevi uma espécie de programa simples para selecionar entre obter o clima hora a hora (opcao 1) ou atual (opcao 2). Se o utilizador introduzir cualquer outro numero ou nenhum, a consola da erro. O comando lembrou-me bastante ao *switch case* no Python.
### 5- Configurando a primeira opcao, dei me conta de que mesmo com o pequeno programa a funcionar, devolvia outputs só em duas linhas e difíceis de ler. Pesquisei uma forma de mudar a formatacao do output e encontrei o comando *PSCustomObject* para criar objetos, mas ainda assim a tabela só tinha uma linha em cada coluna.
### 6- Após testar, descobri que a razao pela qual cada coluna tinha uma só linha era porque todos os valores eram retornados de uma só vez, mas por exemplo o objeto *$resultado.hourly.time[0]* devolvia só a hora 00:00. Usando esta lógica, pesquisei se haveriam condicoes no powershell tal como no python ou no c++, para fazer um loop e retornar uma hora por linha até o número de loops ser igual a 24 (quantidade de horas no dia).
### 7- Encontrei o comando *for*, que é exatamente igual ao comando do mesmo nome na linguagem c++, mas com uma pequena mudanca na parte das condicoes (deve ser utilizado -lt invés de <). Utilizei esse comando, com os parametros *$a=0; $a -lt 24; $a++*, para indicar o inicio de um loop que iria retornar os valores 1, 2, 3, e assim sucessivamente até o valor 23.
### 8- Configurei o *PSCustomObject* para ter duas colunas: Horas, que iria retornar o valor do campo correspondente ao valor de $a (ou seja, no primeiro loop iria retornar o valor do campo zero ou inicial que é 00:00, no segundo o valor do primeiro campo que é 01:00, no terceiro 02:00, e assim sucessivamente), e a coluna Temperatura em Celsius, que iria retornar a temperatura no campo correspondente ao valor de $a. Ficou dessa maneira uma tabela mais favorável na vista.
### 9- Configurei o caso 2, ou seja, a temperatura e data atuais, que foi muito mais simples. Criei um *PSCustomObject* com duas colunas: Dia e hora atuais, que iria retornar o tempo atual como indicado na api, e Temperatura atual em Celsius, que iria retornar a temperatura atual.
