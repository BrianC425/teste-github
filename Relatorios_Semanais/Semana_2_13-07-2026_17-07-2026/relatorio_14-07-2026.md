# Hoje, dia 14/07/2026, criei dois containers no Docker: um frontend e um backend.

### -Investiguei sobre alguns conceitos requeridos para o exercício, nomeadamente a definição de Docker Compose (uma ferramenta que inicia vários containers ao mesmo tempo, necessária neste exercício para iniciar o container frontend e o container backend)
### -Criei uma pasta **teste** e dentro outras duas pastas, uma chamada **frontend** e outra chamada **backend**.
### -Utilizei **Claude AI** para perceber os seguintes passos, pois nunca tinha trabalhado com Node.js: seguindo o tutorial gerado, criei um ficheiro ***server.js*** na pasta **backend**, e dentro escrevi o código simples gerado pela ferramenta. Este ficheiro é o servidor que simplesmente responde com uma mensagem simples quando alguém visita /api/hello. Foram escritos vários comandos no ficheiro como *express* (uma libraria que facilita o processo de escrever servidores web), *cors* (que permite ao frontend chamar ao backend), e *app.listen(PORT, "0.0.0.0", ...)* (uma regra que indica ao Docker que deve aceitar conexões de qualquer lugar e não só de dentro do container)
### -Depois, criei o ficheiro ***package.json*** na mesma pasta **backend**, o qual indica ao Node.js o nome do projeto e as librarias necessárias. Ao criar o container, Docker lê este ficheiro e instala as librarias indicadas.
### -Como último ficheiro da pasta **backend** criei o ***Dockerfile***, utilizado pelo Docker para criar a imagem do container, neste caso escutando na porta 5000 (*EXPOSE 5000*)
### -Continuando agora com a pasta **frontend**, criei um ficheiro ***index.html*** na mesma, uma página web simples que pede dados ao backend (*http://localhost:5000/api/hello*) e os mostra no ecra. Neste caso mostra a mensagem "*Hello from the Node.js backend!*" pois essa é a mensagem que mostra o container do backend previamente criado.
### -Depois criei o ficheiro ***Dockerfile*** na mesma pasta **frontend**. Neste caso, o container criado a partir da imagem gerada escuta na porta 80 (*EXPOSE 80*).
### -Na pasta **teste**, onde estão as pastas **frontend** e **backend** respetivamente, criei o ficheiro ***docker-compose.yml***, que cria os dois container ao mesmo tempo.
### -De seguida, iniciei um cmd na pasta **teste** e introduzi o comando *docker compose up --build*, que permite ao docker começar com a criação dos respetivos containers.
### -Ao visitar a página *http://localhost:8080*, é possível verificar o funcionamento correto da página Frontend e a mensagem proveniente do backend; similarmente, ao visitar a página *http://localhost:5000/api/hello*, é possível visualizar o JSON do backend diretamente.
